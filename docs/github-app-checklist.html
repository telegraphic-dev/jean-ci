GitHub App checklist

Permissions and events that jean-ci actually needs.

When PR checks don’t show up, the root cause is usually missing app permissions, missing events, or installing the app on the wrong repositories. Use this page as the reality check.

Required repository permissions

PermissionLevelWhy jean-ci needs it
ChecksRead & writeCreate and update check runs for the built-in review and custom prompt checks.
ContentsReadFetch .jean-ci/pr-checks/*.md and .jean-ci/coolify.yml from the repo.
Pull requestsReadRead PR title, body, state, and diff metadata.
MetadataReadRepository identity, installation scope, and basic repo details.

Webhook events to subscribe to

Event list
pull_request
pull_request_review
issue_comment
installation
installation_repositories
registry_package

Failure pattern to avoid

If issue_comment is missing, /review comments will do nothing. If registry_package is missing, deployment automation will look dead even though PR reviews still work.

Install verification

After creating or updating the GitHub App, verify from the CLI before blaming jean-ci.

Verify app identity
gh api /app

What to check manually

  • The app is installed on the repo you are testing.
  • The installation has access to the target repo, not just the org.
  • Permissions match the table above.
  • Webhook deliveries are reaching jean-ci.
If checks never appear, start with the App config before digging into prompt files.
