What you get with jean-ci

jean-ci does two jobs well: PR review orchestration and Coolify deployment triggering. The docs focus on those two paths, with copy-ready examples that work in real repos.

Always-on built-in review

The default jean-ci / Code Review check runs whenever PR reviews are enabled for a repo. It is the baseline review, not an optional extra.

Additive repo-specific checks

Files in .jean-ci/pr-checks/*.md add more checks. They never replace the built-in review. Use them for domain-specific rules like migrations, docs, or security.

Operator-friendly deployment mapping

Use .jean-ci/coolify.yml to map GHCR packages to Coolify app UUIDs. jean-ci reacts to registry publishes and asks Coolify to deploy the already-built image.

Default review behavior, plainly stated

  • Built-in review: always runs when PR review is enabled.
  • Custom checks: files in .jean-ci/pr-checks/ are optional and additive.
  • PR comment: the built-in review also posts the visible PR review comment.
  • Prompt validation: malformed custom prompts fail fast instead of producing mushy LLM output.
