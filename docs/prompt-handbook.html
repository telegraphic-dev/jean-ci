Prompt authoring handbook

Each prompt should be deterministic enough that the same prompt + same diff produces the same verdict style.

Required schema

# Prompt title

## Purpose
One paragraph explaining what this check protects.

## Review Instructions
Concrete things to inspect in the PR diff.

## Verdict Criteria
- **FAIL** when ...
- **PASS** when ...

Good prompt habits

  • Fail only for merge-blocking issues.
  • Scope the check to one concern area.
  • Describe PASS and FAIL explicitly.
  • Avoid vague asks like “review everything”.

Starter prompts

  • Security review
  • Tests and docs coverage
  • Migration safety
