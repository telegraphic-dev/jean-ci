Quickstart

Get from zero to your first jean-ci review.

This path assumes you want the built-in review running first, then one extra repo-specific check. Each step includes what to paste and what success looks like.

Start the app Know defaults Add a check Trigger review

1. Start jean-ci locally or in your environment

Use the app normally first. Don’t add repo prompts until the base app health and GitHub App wiring are working.

Start the app
cp .env.example .env
npm ci
npm run build
npm run start
Verify health
curl -s http://localhost:3000/api/health

Expected result

A healthy response such as {"ok":true}.

2. Know what runs by default

The built-in jean-ci / Code Review check is the baseline review. If PR review is enabled for a repo, that check runs even if the repo has no .jean-ci/pr-checks/ folder yet.

What custom prompts do

Files in .jean-ci/pr-checks/*.md only add extra checks. They do not replace the built-in review or the built-in PR review comment.

3. Add one repo-specific check

This creates an extra security check in addition to the default Code Review.

Create .jean-ci/pr-checks/security.md
mkdir -p .jean-ci/pr-checks
cat > .jean-ci/pr-checks/security.md <<'EOF'
# Security Review

## Purpose
Catch merge-blocking security regressions before they land.

## Review Instructions
Review the PR diff for exposed secrets, missing authorization checks, unsafe input handling, injection risk, path traversal, and broken trust boundaries.
Reference the changed code paths when you find a blocking issue.
Only fail for real merge-blocking problems.

## Verdict Criteria
- **FAIL** when the diff introduces a real vulnerability, exposes secrets, or removes a required authorization or validation guard.
- **PASS** when no merge-blocking security issues are present in the diff.
EOF
Commit the prompt
git add .jean-ci/pr-checks/security.md
git commit -m "chore: add jean-ci security review"

4. Trigger a review

jean-ci runs on PR open, synchronize, ready-for-review, and explicit /review comments.

Force a rerun on an existing PR
gh pr comment <pr-number> --body "/review"

Expected result

  • One always-on jean-ci / Code Review check
  • One additional jean-ci / security.md check
  • A visible PR review comment created by the built-in Code Review

5. OpenClaw-ready prompt to ask an agent for the setup

If you want OpenClaw to generate the repo-side prompt files for you, paste this directly into your agent chat.

OpenClaw agent prompt
Set up jean-ci PR reviews in this repository.

Requirements:
- Keep the built-in default `jean-ci / Code Review` enabled and untouched.
- Add one additive custom check at `.jean-ci/pr-checks/security.md`.
- Use the structured jean-ci prompt format with `## Purpose`, `## Review Instructions`, and `## Verdict Criteria`.
- Make the custom check fail only for merge-blocking security issues.
- Commit the changes on a fresh branch and open a PR.
- In the PR summary, explicitly state that `.jean-ci/pr-checks/*` is additive and does not replace the built-in review.
If you only remember one thing: custom checks are additive; the built-in Code Review is the always-on default.
