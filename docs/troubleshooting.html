Troubleshooting

Fast answers when jean-ci looks broken.

Most failures fall into four buckets: app permissions, malformed prompt files, dead gateway auth, or wrong assumptions about what should run by default.

Symptom → likely cause → fix

SymptomLikely causeFix
No checks appear on the PRGitHub App missing permissions, missing installation access, or PR review not enabled for the repo.Check app permissions, repo installation scope, and repo review settings first.
Only the built-in review appearsNo .jean-ci/pr-checks/*.md files exist, or they were added on a different branch/commit than the PR head.Add or fix the prompt files on the PR branch. The built-in review appearing alone is normal.
Custom check fails immediately with prompt errorThe prompt file is missing the required structured sections or explicit PASS/FAIL criteria.Rewrite the prompt to include title, Purpose, Review Instructions, and Verdict Criteria.
Review says execution failedGateway URL/token is missing, invalid, or the gateway is unreachable.Verify env vars and test the gateway health endpoint directly.
/review comment does nothingissue_comment webhook event is not enabled, or the comment text is wrong.Enable issue_comment and use the exact /review command.

Important mental model

The built-in jean-ci / Code Review is the default review. Custom prompt files are additive. If you expected custom files to replace the built-in review, that expectation is the bug.

When prompt output looks odd

Keep custom prompts narrow and explicit. The more a prompt tries to be a general reviewer, the noisier and less trustworthy it gets.

Troubleshooting works better when the docs tell you the intended behavior first. That’s why the default/additive split is repeated across the site.
